```text

                       REST API (HTTPS)

                              │

                    Express.js Backend Server

          User Management
          Business Logic
          AI Services
          Resume Services
          Progress Tracking

                              │

                     Supabase PostgreSQL

      Tables
      Views

```

# 8. Row Level Security

Every table containing user data must enable Row Level Security (RLS).

Policies:

Users can only read their own records.

Users can only update their own records.

Users cannot access other users' data.

Public reference tables (skills_master, career_insights, learning_resources, interview_questions) are readable by all authenticated users.

# 11. Security Best Practices

Use JWT authentication via Supabase Auth.

Store passwords only through Supabase Auth (never in application tables).

Validate all request payloads before processing.

Use parameterized queries to prevent SQL injection.

Enable HTTPS for all API traffic.

Apply rate limiting to authentication endpoints.

Use environment variables for all secrets and API keys.

Implement audit logging for sensitive operations.
